# Requirement ID: FR1

- Description: [The system shall provide users with immediate access to supportive conversation flows and coping exercises when they report feeling anxious or overwhelmed.]
- Source Persona: [Anxious Student Seeking Emotional Support]
- Traceability: [Derived from review group G1]
- Acceptance Criteria: [Given the user indicates that they are feeling anxious or overwhelmed When the support request is submitted Then the application must present at least one relevant coping exercise or supportive response without requiring the user to navigate away.]

# Requirement ID: FR2

- Description: [The system shall maintain a private and judgment-free interaction environment for users seeking emotional support.]
- Source Persona: [Anxious Student Seeking Emotional Support]
- Traceability: [Derived from review group G1]
- Acceptance Criteria: [Given the user is interacting with the chatbot about stress or anxiety When responses are generated Then the responses must remain supportive, non-judgmental, and focused on emotional assistance.]

# Requirement ID: FR3

- Description: [The system shall provide sleep and relaxation content that users can access before bedtime.]
- Source Persona: [Sleep-Deprived User Seeking Relaxation]
- Traceability: [Derived from review group G2]
- Acceptance Criteria: [Given the user opens the application at night looking for sleep help When the user selects a sleep or relaxation feature Then the application must provide accessible calming content such as guided exercises, sleep support, or relaxation activities.]

# Requirement ID: FR4

- Description: [The system shall minimize required interaction steps for users accessing sleep support features.]
- Source Persona: [Sleep-Deprived User Seeking Relaxation]
- Traceability: [Derived from review group G2]
- Acceptance Criteria: [Given the user wants to start a sleep-support activity When the user begins from the home screen Then the desired content must be reachable in no more than three user actions.]

# Requirement ID: FR5

- Description: [The system shall provide structured journaling and self-reflection exercises for users who want to reflect on their thoughts and emotions.]
- Source Persona: [Reflective User Practicing Self-Improvement]
- Traceability: [Derived from review group G3]
- Acceptance Criteria: [Given the user wants to write about their thoughts or feelings When the journaling feature is opened Then the application must provide a structured prompt, reflection space, or guided exercise.]

# Requirement ID: FR6

- Description: [The system shall support CBT-style exercises that help users examine and reframe negative thoughts.]
- Source Persona: [Reflective User Practicing Self-Improvement]
- Traceability: [Derived from review group G3]
- Acceptance Criteria: [Given the user selects a CBT-related activity When the activity begins Then the application must guide the user through at least one structured step for identifying or reframing thoughts.]

# Requirement ID: FR7

- Description: [The system shall clearly distinguish between free features and subscription-based premium features.]
- Source Persona: [Price-Sensitive User Frustrated by Paywalls]
- Traceability: [Derived from review group G4]
- Acceptance Criteria: [Given the user browses available features When premium-only functionality is displayed Then the application must clearly label those features before the user attempts to access them.]

# Requirement ID: FR8

- Description: [The system shall provide users with clear subscription and pricing information before any purchase decision is required.]
- Source Persona: [Price-Sensitive User Frustrated by Paywalls]
- Traceability: [Derived from review group G4]
- Acceptance Criteria: [Given the user views a premium feature or subscription screen When pricing information is shown Then the application must display the subscription cost, billing model, and premium access details in the same view.]

# Requirement ID: FR9

- Description: [The system shall allow users to access core application functions without crashes, freezes, or unexpected termination.]
- Source Persona: [Frustrated User Facing Technical Issues]
- Traceability: [Derived from review group G5]
- Acceptance Criteria: [Given the user is actively using a core feature When the interaction is completed Then the application must remain responsive and must not crash or freeze.]

# Requirement ID: FR10

- Description: [The system shall allow users to log in and access previously available features reliably.]
- Source Persona: [Frustrated User Facing Technical Issues]
- Traceability: [Derived from review group G5]
- Acceptance Criteria: [Given the user enters valid login credentials When the login request is submitted Then the application must authenticate the user successfully and open the main interface without error.]
